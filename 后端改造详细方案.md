# ScrollWeaver 后端改造详细方案

## 一、当前架构分析

### 1.1 现有技术栈
- **Web框架**：FastAPI
- **数据库**：SQLite（通过 `database.py` 管理）
- **WebSocket**：Socket.IO（通过 `socketio_manager.py` 管理）
- **会话管理**：`SessionManager`（支持CHAT、STORY、GAME三种模式）
- **核心模块**：
  - `modules/orchestrator.py` - 世界编导
  - `modules/main_performer.py` - 角色表演者
  - `modules/chat/chat_performer.py` - 私语模式
  - `modules/werewolf/` - 狼人杀游戏
  - `modules/business/` - 商业博弈游戏

### 1.2 现有数据库表结构
- `users` - 用户表
- `scrolls` - 书卷表
- `stories` - 故事表
- `user_tokens` - 用户token表
- `business_games` - 商业博弈游戏记录表
- `multiplayer_rooms` - 联机房间表
- `multiplayer_room_players` - 联机房间玩家表

### 1.3 现有API端点分类
- 用户认证：`/api/login`, `/api/register`, `/api/logout`
- 书卷管理：`/api/create-scroll`, `/api/list-scrolls`, `/api/get-scroll`
- 私语模式：`/api/chat/create`, `/api/chat/send`, `/api/chat/history`
- 故事模式：`/api/story/create`, `/api/story/send`
- 游戏模式：`/api/werewolf/*`, `/api/business/*`, `/api/who-is-human/*`

**核心功能**：

**类设计**：
```python
class TimeSystem:
    def __init__(self, session_id: str, scroll_id: int, db: Database):
        """
        初始化时间系统
        
        Args:
            session_id: 世界会话ID
            scroll_id: 书卷ID
            db: 数据库实例
        """
        self.session_id = session_id
        self.scroll_id = scroll_id
        self.db = db
        self.session_data = None  # 从数据库加载会话数据
        
    def get_current_world_time(self) -> datetime:
        """
        获取当前世界时间
        
        计算逻辑：
        - 如果用户在线且未暂停：
          current_world_time = paused_world_time + (current_real_time - last_online_time) * time_speed
        - 如果用户离线或已暂停：
          current_world_time = paused_world_time
        """
        
    def pause_time(self) -> bool:
        """暂停时间（用户离线时调用）"""
        
    def resume_time(self) -> bool:
        """恢复时间（用户重新上线时调用）"""
        
    def set_world_time(self, target_time: datetime) -> bool:
        """强制设置世界时间"""
        
    def set_time_speed(self, speed: int) -> bool:
        """设置时间流速（现实1秒=书中N分钟）"""
        
    def check_time_events(self) -> List[Dict]:
        """检查并触发时间相关事件"""
        
    def format_world_time(self, world_time: datetime) -> str:
        """格式化世界时间显示"""
```

**时间计算逻辑**：
```python
# 伪代码
def calculate_world_time():
    session = db.get_world_session(session_id)
    
    if session['is_paused']:
        return parse_datetime(session['paused_world_time'])
    
    # 计算时间差
    last_online = parse_datetime(session['last_online_time'])
    current_real = datetime.now()
    real_time_diff = (current_real - last_online).total_seconds()
    
    # 转换为世界时间
    world_time_diff = real_time_diff * session['time_speed']  # 秒转分钟
    
    # 计算当前世界时间
    paused_world_time = parse_datetime(session['paused_world_time'])
    current_world_time = paused_world_time + timedelta(minutes=world_time_diff)
    
    return current_world_time
```

#### 4.2.1 新增 `WorldSession` 类

```python
class WorldSession(BaseSession):
    """
    世界会话：统一管理用户在世界中的体验
    """
    
    def __init__(self, session_id: str, scroll_id: int, user_id: int,
                 cross_type: str, **kwargs):
        super().__init__(session_id, scroll_id, user_id, SessionMode.WORLD)
        self.cross_type = cross_type  # 'character' | 'self' | 'soulverse'
        self.character_code = kwargs.get('character_code')
        self.persona_model_id = kwargs.get('persona_model_id')
        
        # 世界视图实例
        self.world_view = None
        
        # 当前活跃的子会话（私语、入卷、雅集）
        self.active_chat_session = None
        self.active_story_session = None
        self.active_game_session = None
        
    async def initialize(self, config: Dict[str, Any]) -> Dict[str, Any]:
        """初始化世界会话"""
        from modules.world.world_view import WorldView
        
        # 创建世界视图
        self.world_view = WorldView(
            session_id=self.session_id,
            scroll_id=self.scroll_id,
            user_id=self.user_id,
            db=db
        )
        
        # 初始化世界视图
        init_result = await self.world_view.initialize(
            cross_type=self.cross_type,
            character_code=self.character_code,
            persona_model_id=self.persona_model_id
        )
        
        return {
            "status": "initialized",
            "session_id": self.session_id,
            "world_status": init_result
        }
    
    async def process_message(self, message: Dict[str, Any], sender_id: int) -> Dict[str, Any]:
        """处理世界消息"""
        action_type = message.get("action_type")
        
        if action_type == "move":
            # 移动地点
            location_code = message.get("location_code")
            return await self.world_view.move_to_location(location_code)
            
        elif action_type == "start_chat":
            # 启动私语
            character_code = message.get("character_code")
            return await self.world_view.start_chat(character_code)
            
        elif action_type == "start_story":
            # 启动入卷
            return await self.world_view.start_story()
            
        elif action_type == "start_game":
            # 启动雅集游戏
            game_type = message.get("game_type")
            return await self.world_view.start_game(game_type)
            
        elif action_type == "update_time":
            # 更新时间（强制更改时间）
            target_time = message.get("target_time")
            return await self.world_view.time_system.set_world_time(target_time)
            
        elif action_type == "toggle_ai_mode":
            # 切换AI代理模式（灵魂降临）
            return await self._toggle_ai_mode()
    
    async def cleanup(self):
        """清理世界会话"""
        # 保存世界状态
        if self.world_view:
            self.world_view.save_state()
        
        # 清理子会话
        if self.active_chat_session:
            await self.active_chat_session.cleanup()
        if self.active_story_session:
            await self.active_story_session.cleanup()
        if self.active_game_session:
            await self.active_game_session.cleanup()
```

### 4.4 故事导出模块

#### 4.4.1 `modules/world/story_exporter.py`

```python
class StoryExporter:
    def __init__(self, session_id: str, db: Database):
        self.session_id = session_id
        self.db = db
        
    def export_story(self) -> str:
        """
        导出故事为TXT格式
        
        返回：TXT格式的故事文本
        """
        # 1. 获取会话信息
        session = self.db.get_world_session(self.session_id)
        
        # 2. 获取所有交互记录
        interactions = self.db.get_world_interactions(self.session_id)
        
        # 3. 按时间排序
        interactions.sort(key=lambda x: x['world_time'])
        
        # 4. 生成TXT内容
        txt_content = self._generate_txt_content(session, interactions)
        
        return txt_content
    
    def _generate_txt_content(self, session: Dict, interactions: List[Dict]) -> str:
        """生成TXT内容"""
        lines = []
        
        # 标题
        lines.append("=" * 60)
        lines.append(f"世界穿越记录：{session['scroll_id']}")
        lines.append("=" * 60)
        lines.append("")
        
        # 基本信息
        lines.append(f"开始时间：{session['created_at']}")
        lines.append(f"穿越方式：{session['cross_type']}")
        if session['character_code']:
            lines.append(f"扮演角色：{session['character_code']}")
        lines.append("")
        
        # 交互记录
        lines.append("-" * 60)
        lines.append("交互记录")
        lines.append("-" * 60)
        lines.append("")
        
        for interaction in interactions:
            # 时间戳
            lines.append(f"[{interaction['world_time']}]")
            
            # 交互类型和内容
            lines.append(f"类型：{interaction['interaction_type']}")
            if interaction['location_code']:
                lines.append(f"地点：{interaction['location_code']}")
            
            # 内容
            lines.append(interaction['content'])
            lines.append("")
        
        return "\n".join(lines)
```

## 九、关键技术点

### 9.1 时间系统实现

**关键问题**：
1. **时间精度**：使用 `datetime` 还是自定义时间类型？
2. **时区处理**：是否需要考虑时区？
3. **时间事件触发**：如何高效检查时间事件？

**解决方案**：
- 使用 `datetime` 存储世界时间（ISO格式字符串）
- 时间计算使用 `timedelta`
- 时间事件使用定时任务或轮询检查

### 9.2 世界状态持久化

**关键问题**：
1. **状态粒度**：保存哪些状态？
2. **状态合并**：如何处理状态冲突？
3. **性能优化**：如何减少数据库写入？

**解决方案**：
- 状态按类型分类存储（location、event、relationship等）
- 使用 `UNIQUE` 约束避免重复
- 批量更新，减少数据库操作

### 9.3 游戏改造兼容性

**关键问题**：
1. **如何保持原有架构**？
2. **如何无缝切换**？

**解决方案**：
- 不修改游戏核心逻辑
- 在输出层添加格式化方法
- 使用适配器模式连接世界角色和游戏角色
