# ScrollWeaver 产品演进计划

## 一、产品定位

**基于AOP架构的AI沉浸式世界构建引擎**

- **一卷一世界**：上传书卷，提取世界观、人物卡、地理志
- **沉浸式交互**：地图、时间表、多玩法融合
- **多身份体验**：扮演角色、做自己、Soulverse AI代理
- **养成系体验**：长期参与，世界持续演化

#### 2.5.3 时间表系统

**功能描述**：实时显示当前世界时间，影响事件触发和剧情发展

**时间流速**：可配置（默认：现实1秒 = 书中1分钟）

**需要实现**：
- [ ] 创建 `TimeSystem` 类
- [ ] 记录世界开始时间
- [ ] 记录用户最后在线时间
- [ ] **时间暂停机制**：用户离线时，世界时间暂停
- [ ] **时间恢复机制**：用户重新上线时，从暂停时间点继续
- [ ] **强制更改时间**：用户可以手动设置当前世界时间
- [ ] 实时计算当前世界时间（基于时间流速和在线状态）
- [ ] 显示时间表UI（当前时间、日期、季节等）
- [ ] 提供时间流速配置选项
- [ ] 提供强制更改时间的UI和功能
- [ ] 基于时间触发事件（如：特定时间点的事件）
- [ ] 时间影响剧情发展（如：白天/夜晚不同事件）

**数据模型**：
- 在会话中记录：
  - `world_start_time`（现实时间戳）
  - `last_online_time`（用户最后在线时间）
  - `paused_time`（暂停时的世界时间）
  - `time_speed`（时间流速配置，默认60）
  - `is_paused`（是否暂停）
- 计算逻辑：
  - 如果用户在线：`current_world_time = paused_time + (current_real_time - last_online_time) * time_speed`
  - 如果用户离线：`current_world_time = paused_time`（时间暂停）

**后端API**：
- `GET /api/world/{session_id}/time` - 获取当前世界时间
- `PUT /api/world/{session_id}/time` - 强制更改世界时间
- `PUT /api/world/{session_id}/time-speed` - 配置时间流速
- `POST /api/world/{session_id}/events/trigger` - 触发时间相关事件

**后端API**：

## 五、关键设计决策

### 5.1 时间系统
- **时间流速**：可配置（默认：现实1秒 = 书中1分钟）
- **离线暂停**：用户离线时，世界时间暂停
- **时间恢复**：用户重新上线时，从暂停时间点继续
- **强制更改时间**：用户可以手动设置当前世界时间
- **时间存储**：基于现实时间戳和在线状态计算世界时间
- **事件触发**：基于世界时间触发事件

### 5.2 交互触发
- **用户主动选择**：所有玩法都由用户主动触发
- **前置条件检查**：入卷需要多人，雅集需要多人+游戏意愿
- **随时可启动私语**：不受地点人数限制

### 5.3 游戏改造
- **保持原有架构**：不改变游戏核心逻辑和架构，保证运行稳定
- **输出改为文字描述**：游戏过程以文字描述展示，而非可视化界面
- **使用世界角色**：游戏中的角色来自世界人物卡
- **场景描述**：游戏过程中包含当前世界的场景描述

### 5.4 Soulverse模式
- **使用已有系统**：Soulverse系统已存在，直接集成使用
- **AI代理**：AI基于人格模型代替用户反应
- **灵魂降临**：用户可以随时接管控制
